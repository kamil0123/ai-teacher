## How to Contribute

1. **Open an issue first** for significant changes — discuss the approach before
   investing time in code.
2. **Fork** the repository and create a feature branch from `main`.
3. **Write clear, focused commits.** Keep unrelated changes in separate PRs.
4. **Add or update tests** for any changed behaviour.
5. **Open a Pull Request** against `main` with a clear description of what and
   why.

---

## What We Accept

- Bug fixes
- Documentation improvements
- New features that align with the project roadmap

## What We Do Not Accept

- Changes that introduce new dependencies without prior discussion
- Code that violates the security guidelines in the README
- Contributions that infringe third-party intellectual property

---

## Code Style

- Follow the existing code style in the file you are editing.
- Run linting before submitting (`ruff check .` / `ruff format .`).
- All tests must pass (`pytest`).
